# Import and Type Systems Conventions

Цей документ визначає суворі правила роботи з імпортами, типами та конфігурацією збирача у проєкті Puzzity.

## 1. Заборона нативного `import React`
У зв'язку з використанням React 19 та сучасного JSX-трансформера, імпортувати глобальний об'єкт `React` у файлах компонентів **заборонено**, якщо ви не використовуєте його безпосередньо (наприклад, `React.Component`).
- **❌ Неправильно:** `import React, { useState } from 'react'`
- **✅ Правильно:** `import { useState } from 'react'`

## 2. Суворі Type-Only імпорти (`verbatimModuleSyntax`)
У нашому проєкті увімкнено параметр компілятора `"verbatimModuleSyntax": true`. Це означає, що типи та інтерфейси TypeScript мають бути імпортовані виключно з використанням ключового слова `type`. Недотримання цього правила призведе до помилки компіляції TS1484.
- **❌ Неправильно:** `import { ReactNode, HTMLAttributes } from 'react'`
- **✅ Правильно:** `import type { ReactNode, HTMLAttributes } from 'react'`

Якщо ви імпортуєте водночас і функції, і типи з одного файлу, використовуйте інлайнові type-імпорти:
- **✅ Правильно:** `import { useState, type ReactNode } from 'react'`

## 3. Очищення від невикористаних імпортів (`noUnusedLocals`)
Правило компілятора `noUnusedLocals` є увімкненим. Будь-який імпортований елемент, який не використовується в коді файлу, викликає помилку збірки. Перед кожним комітом перевіряйте відсутність "сірих" невикористаних імпортів.

## 4. Абсолютні шляхи (Aliases)
Завжди використовуйте абсолютні аліаси `@/` для імпортів поза межами вашої поточної папки. Відносні шляхи типу `../../` дозволені виключно для файлів, які знаходяться в одній локальній папці (сусідні файли).
- **❌ Неправильно:** `import { Button } from '../../shared/ui/Button'`
- **✅ Правильно:** `import { Button } from '@/shared/ui/Button'`
