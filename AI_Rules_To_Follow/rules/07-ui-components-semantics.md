# UI Components & Semantics

Цей документ визначає суворі правила розробки UI-компонентів, семантики та форматування даних у Puzzity.

## 1. Поліморфна Семантика (SEO & A11y)
Візуальний вигляд тексту та його HTML-тег повинні бути повністю розділені.
- **❌ Заборонено:** `<div className="text-h1">Title</div>`
- **✅ Дозволено:** `<Heading as="h1" variant="h3">Title</Heading>`

## 2. Форматування Даних (Intl API)
Ніколи не форматуйте дати чи валюти вручну (через конкатенацію рядків). 
- **❌ Заборонено:** `<span>{price} ₴</span>` або `<span>{date.split('-')[0]}</span>`
- **✅ Дозволено:** Використовуйте функції з `shared/lib/formatters` та компоненти `<FormattedData>`. Вони автоматично обгортають значення у правильні семантичні теги `<data>`, `<output>` або `<time>`.

## 3. Container-Focus Pattern для Форм
Нативні інпути (`<input>`, `<select>`) не повинні мати власного `outline` або `border`. Усі візуальні стани (`focus`, `error`) відслідковуються на рівні батьківського контейнера (через `:focus-within`), щоб уникнути проблеми квадратних рамок при заокруглених контейнерах.

## 4. Zero-Shift Buttons
Стан завантаження кнопки (`isLoading`) не повинен змінювати її ширину. Текст стає прозорим (`opacity: 0`), а спінер позиціонується абсолютно. Це гарантує відсутність layout shift.

## 5. Доступність Clickable Cards (Overlay Pattern)
Якщо картка має поводитися як чекбокс, але при цьому дозволяти виділяти текст опису або містити посилання:
- Не обгортайте всю картку в `<label>`.
- Обгорніть у `<label>` лише заголовок і додайте до нього `::after { position: absolute; inset: 0; z-index: 1 }`. Всі інші інтерактивні елементи повинні мати `z-index: 2`.
